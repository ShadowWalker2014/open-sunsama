## What is Linear?

Linear is a modern issue tracking and project management tool built specifically for software development teams. Founded in 2019 by former Uber engineers, Linear has rapidly become the preferred issue tracker for startups and engineering teams who've grown frustrated with Jira's complexity.

### How Linear Works

Linear organizes work into a clear hierarchy: **Workspace ‚Üí Teams ‚Üí Projects ‚Üí Issues**. Each issue has standard properties‚Äîassignee, priority, status, labels‚Äîbut Linear's magic is in how it makes working with issues feel fast and frictionless.

Everything in Linear is keyboard-accessible. Press `C` to create an issue, `D` to change status, `/` to search. The interface responds instantly. There's no page reloads, no lag, no waiting. For developers accustomed to terminal workflows, Linear feels like a native extension of how they already work.

### Linear's Key Features for Engineers

**Cycles and Roadmaps**: Linear organizes work into time-boxed cycles (like sprints) while maintaining a long-term roadmap view. This bridges the gap between tactical daily work and strategic product planning.

**GitHub/GitLab Integration**: Link PRs to issues automatically. When you push a branch named `feat/LIN-123-add-auth`, Linear connects it to issue LIN-123. Merge the PR, and the issue auto-closes.

**Triage and Inbox**: New issues flow into a triage queue where leads can quickly assign priority and ownership. The inbox shows issues that need your attention without overwhelming notifications.

**Custom Views and Filters**: Build views showing exactly what you need‚Äîyour issues this cycle, high-priority bugs, unassigned work. Save views for quick access.

**Keyboard-First Design**: Nearly every action has a keyboard shortcut. Power users rarely touch the mouse. Command palette (Cmd+K) gives quick access to everything.

### What Linear Doesn't Do

Linear is exceptional at issue tracking, but it's not a personal productivity tool:

**No Daily Planning**: Linear shows what issues exist, not what you should work on today. There's no ritual for deciding your daily priorities.

**No Time Blocking**: You can't schedule issues onto your calendar or visualize how your day should flow.

**No Focus Mode**: Linear doesn't help you actually do the work‚Äîit helps you track the work.

**No Personal Task Management**: Linear is for team issues, not your personal to-dos (pick up groceries, schedule dentist, review that blog post).

This is why many engineers using Linear also need a separate daily planning tool.

## What is Sunsama?

Sunsama is a daily planner designed for knowledge workers who want to be intentional about how they spend their time. It's not an issue tracker‚Äîit's a tool for deciding "what will I accomplish today?" and then actually doing it.

### How Sunsama Works

Sunsama centers on a daily planning ritual. Each morning, you:

1. **Review your calendar**: See meetings and commitments that already claim your time
2. **Pull in tasks**: Import tasks from Linear, GitHub, Trello, Asana, or other tools
3. **Set intentions**: Consciously decide what deserves your time today
4. **Time block**: Drag tasks onto specific calendar slots
5. **Check workload**: Sunsama warns if you've overcommitted

Throughout the day, you work through your time blocks. Focus mode expands the current task to fill your screen, starts a timer, and minimizes distractions.

### Sunsama's Key Features for Developers

**Linear Integration**: Sunsama has native integration with Linear. Pull your assigned issues into your daily planning view without leaving Sunsama. When you complete a task, it can update the Linear issue.

**GitHub Integration**: Pull GitHub issues and PRs into your daily workflow. See what needs review, what's ready to merge, and plan accordingly.

**Focus Mode**: When you start working on a task, focus mode creates a distraction-free environment. Pomodoro timer support for structured work sessions.

**Calendar Sync**: Two-way sync with Google Calendar and Outlook. Your time blocks appear on your calendar, preventing others from booking over your deep work time.

**Daily Shutdown**: End-of-day ritual helps you reflect on accomplishments and roll incomplete tasks to tomorrow.

## The Developer Workflow: Using Linear AND a Daily Planner

Here's the key insight: **Linear and daily planners serve different stages of the same workflow**. Comparing them directly misses the point.

### The Complete Engineering Workflow

1. **Product/Team Level (Linear)**: Issues get created, prioritized, and assigned. Cycles define what the team is working on this sprint. Roadmaps show long-term direction.

2. **Personal Planning Level (Daily Planner)**: Each morning, you pull your assigned Linear issues into your daily view. You see your meetings, estimate focus time available, and decide which issues get attention today.

3. **Execution Level (Focus Mode)**: With your day planned, you work through time blocks. Focus mode helps you actually complete the work without distractions.

4. **Completion Loop (Linear)**: Finished work flows back to Linear‚Äîissues close, PRs merge, the team has visibility into progress.

Most engineers using only Linear skip step 2‚Äîthey just look at their issue list and start working on whatever seems important. This leads to:
- Context switching between many issues
- Underestimating how much is realistic for one day
- Missing the deep work time needed for complex issues
- Feeling busy but not productive

Adding intentional daily planning bridges this gap.

## Why Engineers Need BOTH Issue Tracking AND Daily Planning

### The Case for Linear

Linear excels at team coordination:
- **Shared context**: Everyone sees what's being worked on
- **History and accountability**: Issues track who did what and when
- **Integration with code**: PRs link to issues automatically
- **Async communication**: Comments and discussions live with the work
- **Prioritization at scale**: Cycles and roadmaps organize hundreds of issues

You can't run an engineering team without issue tracking. Linear (or Jira, or GitHub Issues) is non-negotiable.

### The Case for Daily Planning

Issue trackers show what work exists. Daily planners answer a different question: "What will I actually accomplish today?"

**Engineers need daily planning because:**

1. **Focus time is scarce**: With meetings, Slack, and interruptions, you might only have 4-5 hours of focus time. Planning helps you protect and use it wisely.

2. **Issues vary in size**: A "quick bug fix" might take 30 minutes; a "refactor auth module" might take 3 days. Daily planning forces you to be realistic about what fits today.

3. **Energy management matters**: Creative problem-solving requires different energy than code review. Plan deep work for your peak hours.

4. **Personal tasks exist**: Engineers have lives outside of Linear‚Äîplanning helps integrate personal and professional commitments.

5. **Intentionality beats reactivity**: Without planning, you respond to whatever's loudest. With planning, you choose what's most important.

### What's the difference between Linear and Sunsama?

**Linear** is an issue tracking tool for software development teams.
